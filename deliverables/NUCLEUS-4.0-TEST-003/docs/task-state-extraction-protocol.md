# 任务状态提取协议 v1.0

## 1. 协议概述

本协议定义了如何从 TASK-CARD 文件中准确提取任务状态标记，确保全盘扫描机制的可靠性。

## 2. 状态标记定义

| 状态标记 | 含义 | 优先级 |
|----------|------|--------|
| `[x]` | 已完成 | 最高 |
| `[!]` | Harold 待决 | 高 |
| `[?]` | 阻塞 | 高 |
| `[V]` | 待验收 | 中 |
| `[P]` | 执行中 | 中 |
| `[ ]` | 待接收 | 默认 |

## 3. 提取规则

### 3.1 主要提取位置

**优先级 1：Zone B 运行时状态区域**
- 查找表格中的 `**状态标记**` 字段
- 正则表达式：`\|\s*\*\*状态标记\*\*\s*\|\s*(\[[ xPV?!]\])\s*\|`

**优先级 2：Zone A 或文件底部状态声明**
- 查找包含 "状态：" 的行
- 正则表达式：`\*状态：\`(\[[ xPV?!]\])\``

**优先级 3：全局搜索（最后手段）**
- 按优先级顺序搜索所有状态标记
- 返回找到的第一个（最高优先级）标记

### 3.2 上下文验证

为避免误匹配示例或说明文本中的状态标记，进行上下文验证：
- 标记前后 100 字符内应包含"状态"、"Status"等关键词
- 或标记位于表格、列表等结构化内容中

### 3.3 默认值处理

- 如果未找到任何状态标记，默认返回 `[ ]`（待接收）
- 如果找到多个相同优先级的标记，返回最后一个（最新状态）

## 4. 实现参考

参考实现见：`full_scan.py` 中的 `extract_state_marker()` 函数

```python
def extract_state_marker(content: str) -> str:
    """
    从 TASK-CARD 内容中提取状态标记
    优先级：[x] > [!] > [?] > [V] > [P] > [ ]
    """
    # 方法 1：查找 Zone B 中的状态标记
    state_pattern = r'\|\s*\*\*状态标记\*\*\s*\|\s*(\[[ xPV?!]\])\s*\|'
    match = re.search(state_pattern, content)
    if match:
        return match.group(1)
    
    # 方法 2：查找任意状态标记（按优先级）
    for marker in ["[x]", "[!]", "[?]", "[V]", "[P]", "[ ]"]:
        if marker in content:
            # 上下文验证
            idx = content.find(marker)
            if idx != -1:
                context = content[max(0, idx-100):min(len(content), idx+100)]
                if '状态' in context or 'Status' in context:
                    return marker
    
    return "[ ]"  # 默认待接收
```

## 5. 测试用例

### 5.1 正常情况
- **输入**：包含 `**状态标记** | `[P]` 执行中 |` 的 TASK-CARD
- **输出**：`[P]`

### 5.2 多个状态标记
- **输入**：文件中同时存在 `[ ]` 和 `[P]`
- **输出**：`[P]`（执行中优先级高于待接收）

### 5.3 无状态标记
- **输入**：TASK-CARD 中没有状态标记
- **输出**：`[ ]`（默认值）

### 5.4 误匹配防护
- **输入**：TASK-CARD 中包含示例文本 `"例如：[P] 表示执行中"`
- **输出**：根据实际状态标记，忽略示例文本
