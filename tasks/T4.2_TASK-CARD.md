## 二、Task 目标

定义 logs/ 和 executions/ 目录的 JSONL 日志格式：

1. **logs/YYYY-MM-DD.jsonl**：观测日志格式
2. **executions/YYYY-MM-DD.jsonl**：执行日志格式
3. **字段规范**：时间戳、级别、消息、元数据

---

## 三、核心接口

### 3.1 日志格式规范

```json
// logs/YYYY-MM-DD.jsonl
{
  "timestamp": "2026-04-06T12:00:00Z",
  "level": "INFO|WARN|ERROR",
  "message": "日志消息",
  "source": "agent_id",
  "metadata": {
    "cycle_id": "task-...",
    "scope": "task"
  }
}

// executions/YYYY-MM-DD.jsonl  
{
  "timestamp": "2026-04-06T12:00:00Z",
  "action": "create_cycle|apply_adjustment",
  "scope": "task|topic|project|system",
  "cycle_id": "task-...",
  "result": "success|failure",
  "metadata": {}
}
```

## 七、知识沉淀

### 7.1 LESSON-LEARNED

1. **[LL-001]** 日志格式需要规范：使用 JSONL 格式，便于解析和处理
2. **[LL-002]** 日志字段需要包含必要信息：时间戳、级别、消息、元数据
3. **[LL-003]** 日志文件需要按日期分类：logs/ 和 executions/ 目录分别存储不同类型的日志

### 7.2 DECISION-LIBRARY

1. **[DL-001]** 日志格式：使用 JSONL 格式，包含 timestamp、level、message、source 和 metadata
2. **[DL-002]** 日志分类：logs/ 存储观测日志，executions/ 存储执行日志
3. **[DL-003]** 字段规范：时间戳使用 ISO 8601 格式，级别分为 INFO|WARN|ERROR
