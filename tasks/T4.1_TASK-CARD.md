### 3.2 数据结构

```python
class SystemSnapshot:
    """系统快照"""
    def __init__(self):
        self.timestamp: datetime
        self.active_cycles: Dict[str, int]  # scope -> count
        self.log_entries: List[Dict]
        self.execution_entries: List[Dict]
        self.wait_queue_sizes: Dict[str, int]

class Anomaly:
    """异常对象"""
    def __init__(self):
        self.type: str  # 'performance', 'error', 'concurrency'
        self.severity: str  # 'low', 'medium', 'high'
        self.message: str
        self.timestamp: datetime

class AggregatedResult:
    """聚合结果"""
    def __init__(self):
        self.parent_id: str
        self.child_count: int
        self.success_count: int
        self.failure_count: int
        self.aggregated_evidence: List[Dict]
```

## 九、知识沉淀

### 9.1 LESSON-LEARNED

1. **[LL-001]** 监控模块需要严格只读：不写入任何 CycleUnit，避免对系统产生负面影响
2. **[LL-002]** 系统快照需要包含完整信息：包括 active_cycles、log_entries、execution_entries 和 wait_queue_sizes
3. **[LL-003]** 异常检测需要分类处理：按类型和严重程度分类，方便后续处理

### 9.2 DECISION-LIBRARY

1. **[DL-001]** 监控策略：严格只读，不写入任何 CycleUnit
2. **[DL-002]** 系统快照结构：包含 active_cycles、log_entries、execution_entries 和 wait_queue_sizes
3. **[DL-003]** 异常分类：按类型（performance/error/concurrency）和严重程度（low/medium/high）分类
