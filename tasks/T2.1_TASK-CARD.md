## 二、Task 目标

实现持久化调度计数器，支持：

1. **跨 Heartbeat 保持状态**：进程崩溃重启后计数器不丢失
2. **多粒度计数**：task/topic/project/system 四层独立计数
3. **原子写入**：防止并发损坏
4. **阈值触发**：达到阈值自动重置并触发调度

## 三、核心接口

### 3.1 scheduler_state.yaml 结构

```yaml
# cycles/scheduler_state.yaml
last_updated: "2026-04-06T10:45:00Z"
counters:
  task: 0        # 每 1 tick 重置
  topic: 3       # 已运行 3 次，下次第 4 tick 触发
  project: 47    # 已运行 47 次，下次第 48 tick 触发
  system: 1439   # 已运行 1439 次，下次第 336 tick 触发

# 触发阈值
thresholds:
  task: 1      # 每 tick
  topic: 4     # 每 4 tick (2h)
  project: 48  # 每 48 tick (1d)
  system: 336  # 每 336 tick (1w)
```
