# ADR-001 · duty-mapping.yaml vs agent-registry.yaml 同步机制

**ADR ID**: ADR-001  
**主题**: 双配置文件同步机制  
**状态**: 已决策（2026-04-03）  
**决策者**: 张铁 (CQO)  
**评审**: 待 Harold 确认

---

## 一、背景

NUCLEUS-4-0 引入 `agent-registry.yaml` 作为 Agent 全生命周期管理的核心配置文件，但现有系统已使用 `duty-mapping.yaml` 定义 Agent 职责映射。需要明确两者关系，避免 Phase 2 开发时产生冲突。

---

## 二、决策

### 2.1 配置文件定位

| 配置文件 | 定位 | 权威性 | 更新频率 |
|----------|------|--------|----------|
| **agent-registry.yaml** | Agent 全生命周期管理（注册/发现/健康/负载） | ⭐⭐⭐⭐⭐（主） | 动态（运行时更新） |
| **duty-mapping.yaml** | 治理职责映射（静态配置） | ⭐⭐（从） | 静态（部署时更新） |

### 2.2 同步机制

```
┌─────────────────────────────────────────────────────────────┐
│                    配置同步架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  agent-registry.yaml（主配置文件）                           │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ agents:                                               │   │
│  │   CQO:                                                │   │
│  │     name: 张铁                                        │   │
│  │     skills: [governance-quality, governance-delegation]│   │
│  │     status: active                                    │   │
│  │     load: 0.6                                         │   │
│  │     health:                                           │   │
│  │       last_heartbeat: 2026-04-03T15:00:00Z            │   │
│  │       status: healthy                                 │   │
│  └──────────────────────────────────────────────────────┘   │
│                          │                                   │
│                          │ ⭐ 单向同步（只读）                │
│                          ▼                                   │
│  duty-mapping.yaml（从配置文件）                             │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ # 此文件由 agent-registry.yaml 同步生成（只读）        │   │
│  │ mappings:                                             │   │
│  │   CQO:                                                │   │
│  │     responsibilities: [quality, delegation]           │   │
│  │     skills: [governance-quality, governance-delegation]│   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 核心规则

| 规则 | 说明 |
|------|------|
| **同步方向** | agent-registry.yaml → duty-mapping.yaml（单向） |
| **权威性** | agent-registry.yaml 优先（冲突时以此为准） |
| **更新触发** | agent-registry.yaml 变更时自动同步 |
| **只读约束** | duty-mapping.yaml 禁止手动编辑（Phase 2 起） |
| **迁移路径** | Phase 3 合并为单一配置文件 |

## 三、技术实现

### 3.1 同步流程

```python
class AgentRegistrySyncer:
    """
    agent-registry.yaml → duty-mapping.yaml 同步器
    """
    
    def on_agent_registry_changed(self, new_config: dict) -> None:
        # Step 1: 读取 agent-registry.yaml
        agents = new_config.get('agents', {})
        
        # Step 2: 转换为 duty-mapping 格式
        mappings = {}
        for agent_id, agent_info in agents.items():
            mappings[agent_id] = {
                'responsibilities': self._extract_responsibilities(agent_info),
                'skills': agent_info.get('skills', []),
                'synced_at': datetime.utcnow().isoformat()
            }
        
        # Step 3: 写入 duty-mapping.yaml（只读生成）
        duty_mapping = {
            '# Generated from agent-registry.yaml - DO NOT EDIT': True,
            'synced_from': 'agent-registry.yaml',
            'synced_at': datetime.utcnow().isoformat(),
            'mappings': mappings
        }
        self._write_duty_mapping(duty_mapping)
        
        # Step 4: 记录同步日志
        logger.info(f"Synced {len(agents)} agents to duty-mapping.yaml")
```

### 3.2 冲突检测

```python
def detect_conflicts(self, agent_registry: dict, duty_mapping: dict) -> list[Conflict]:
    """
    检测两配置文件冲突（Phase 1 用于迁移验证）
    """
    conflicts = []
    
    for agent_id in agent_registry.get('agents', {}):
        ar_skills = set(agent_registry['agents'][agent_id].get('skills', []))
        dm_skills = set(duty_mapping.get('mappings', {}).get(agent_id, {}).get('skills', []))
        
        if ar_skills != dm_skills:
            conflicts.append(Conflict(
                agent_id=agent_id,
                field='skills',
                agent_registry_value=ar_skills,
                duty_mapping_value=dm_skills,
                resolution='USE_AGENT_REGISTRY'  # 权威性规则
            ))
    
    return conflicts
```

## 五、风险评估

| 风险 | 影响 | 概率 | 应对措施 |
