## 二、§三.3.3 修改对比

### 2.1 当前版本（v2.10.0）

```markdown
### 3.3 异常处理

| 场景 | 处理 |
|------|------|
| Skill 加载失败 | 返回友好提示，降级到 main 处理 |
| 意图歧义 | 取最高置信度，列出备选确认 |
| 完全无法识别 | 降级到闲聊模式 |
```

---

### 2.2 改进后版本（v2.11.0）

```markdown
### 3.3 异常处理（引导式提问） ⭐ v2.11.0

> **核心改进**: 意图不清晰时，提供 A/B/C/D 选项引导用户选择

| 场景 | 处理 |
|------|------|
| Skill 加载失败 | 返回友好提示，降级到 main 处理 |
| **意图歧义** | **引导选择（提供 A/B/C/D 选项 + 记录到 dispatch-state.json）** |
| 完全无法识别 | **引导澄清（提供通用选项模板）** |

### 3.3.1 引导选择流程 ⭐ v2.11.0

```
意图识别 → 多个候选意图
    ↓
读取 dispatch-state.json.user_preferences.last_choice
    ↓
if last_choice 匹配候选意图:
    路由到上次选择的 Skill
else:
    输出引导选项（根据意图类型选择模板）
    ↓
    用户回复 A/B/C/D
    ↓
    记录选择到 dispatch-state.json
    ↓
    路由到目标 Skill
```

### 3.3.2 引导选项模板 ⭐ v2.11.0

**模板选择规则**:

| 候选意图类型 | 使用模板 | 选项数量 |
|--------------|----------|----------|
| 任务相关 | `task_type` | 4 个 |
| 项目相关 | `project_ops` | 4 个 |
| 质量审核 | `quality_ops` | 4 个 |
| 文档相关 | `document_ops` | 4 个 |
| 其他 | 通用澄清模板 | 3 个 |

**模板定义位置**: `templates/dispatch-state.json → guidance_templates`

### 3.3.3 用户偏好记忆 ⭐ v2.11.0

> **核心原则**: 记录用户上次选择，下次相同意图歧义时优先路由到上次选择

**更新字段**:
- `user_preferences.last_intent_type`: 上次意图类型
- `user_preferences.last_choice`: 上次选择（A/B/C/D）
- `user_preferences.choice_history`: 选择历史列表

**示例**:
```json
{
  "user_preferences": {
    "last_intent_type": "create_task",
    "last_choice": "A",
    "choice_history": [
      {"intent": "create_task", "choice": "A", "timestamp": "2026-04-15T09:00:00+08:00"}
    ]
  }
}
```
```
