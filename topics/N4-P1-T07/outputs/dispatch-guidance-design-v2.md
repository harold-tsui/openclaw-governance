> **创建日期**: 2026-04-15

> **用途**: 分层引导设计 + Project/Task 归属验证

---

## 一、Harold 核心约束 ⭐

> **三个"不能"**：
> 1. ❌ 不能出现未知的项目（所有Project必须在 system-projects.yaml 或 user-projects.yaml 中注册）
> 2. ❌ 不能出现孤儿的task card（Task必须有归属Topic）
> 3. ❌ 不能出现孤儿的topic（Topic必须有归属Project）

> **Harold 补充要点（2026-04-15 10:16）** ⭐：
> - **引导触发条件**：不明确归属时才引导，明确归属直接执行
> - **显式声明支持**：用户可说"我在继续进行 ZT-P015 的工作"
> - **任务生命周期管理**：从入口开始管理（创建 → 执行 → 检查 → 完成/阻塞 → 归档）

---

## 二、引导层级设计

### 层级架构

```
┌─────────────────────────────────────┐
│ 层级 1：顶层引导（Project/Task归属验证）│ ← 最重要
├─────────────────────────────────────┤
│ 层级 2：已有Project/Topic选择引导     │ ← 避免"未知项目"
├─────────────────────────────────────┤
│ 层级 3：任务类型引导（如果需要）       │ ← 可选
└─────────────────────────────────────┘
```

---

## 三、层级 1：顶层引导（Project/Task 归属验证） ⭐

### 3.1 用户意图识别

| 用户消息示例 | 意图类型 | 需要引导 |
|--------------|----------|----------|
| "帮我创建一个新项目" | `create_project` | ✅ 顶层引导（是否确实需要新Project） |
| "帮我创建一个新任务" | `create_task` | ✅ 顶层引导（Task归属哪个Project） |
| "帮我处理一下文档" | 意图不清晰 | ✅ 顶层引导（是否需要创建Task） |
| "今天天气怎么样？" | `chat` | ❌ 无需引导 |

---

### 3.2 引导流程

**流程图**：

```
用户消息到达
    ↓
意图识别（是否需要创建Project/Task）
    ↓
if 意图包含"创建":
    ├─ if 意图是 create_project:
    │       引导1：是否确实需要新Project？
    │       ├─ A. 在已有项目基础上创建新Task/Topic（推荐）
    │       ├─ B. 实需要创建新Project（需要Harold审批）
    │       ├─ C. 只是查询/讨论，不需要创建
    │       ↓
    │       用户选择 A → 层级 2（列出已有Project）
    │       用户选择 B → 提示需要Harold审批，暂停创建
    │       用户选择 C → 进入查询/讨论模式
    │
    └─ if 意图是 create_task:
            引导2：Task归属哪个Project？
            ├─ A. 选择已有Project + Topic（推荐）
            ├─ B. 放入临时工作台（ZT-P-AGT-{agent}）
            ├─ C. 放入NUCLEUS治理基础（ZT-P000）
            ↓
            用户选择 A → 层级 2（列出已有Project + Topic）
            用户选择 B/C → 允许创建，但标记为"待归属"
```

---

### 3.3 引导选项模板（顶层）

**模板 1：创建Project引导** ⭐

```
我理解您想创建新项目。请确认：

A. 在已有项目基础上创建新Task/Topic（推荐，避免重复）
B. 确实需要创建新Project（需要Harold审批）
C. 只是查询/讨论，不需要创建

回复 A/B/C 或描述具体需求。
```

**模板 2：创建Task引导** ⭐

```
我理解您想创建新任务。请确认Task归属：

A. 选择已有Project + Topic（推荐）
B. 放入我的临时工作台（ZT-P-AGT-{agent}）
C. 放入NUCLEUS治理基础（ZT-P000）

回复 A/B/C 或描述具体需求。
```

**模板 3：意图不清晰引导** ⭐

```
我理解您有需求，但需要确认是否需要创建任务：

A. 需要创建新任务（会进入Task归属引导）
B. 只需要查询/讨论（不创建Task）
C. 需要执行已有任务（继续未完成的Task）

回复 A/B/C 或描述具体需求。
```

## 四、层级 2：已有Project/Topic选择引导 ⭐

### 4.1 Project列表来源

**数据源**：
- `system-projects.yaml`（系统级Project）
- `user-projects.yaml`（用户级Project）

**排除规则**：
- 排除已关闭的Project（status ≠ active）
- 排除已归档的Project

---

### 4.2 引导流程

**流程图**：

```
用户选择 A（选择已有Project）
    ↓
读取 system-projects.yaml + user-projects.yaml
    ↓
列出活跃Project列表（编号：1/2/3...）
    ↓
用户选择 Project编号
    ↓
读取该Project下的Topic列表
    ↓
列出活跃Topic列表（编号：1/2/3...）
    ↓
用户选择 Topic编号
    ↓
确认创建 → Task归属明确 ✅
```

## 七、验收标准

| 验收项 | 标准 | 验证方法 |
|--------|------|----------|
| **三个"不能"约束** | 引导流程中强制验证 | 流程图审查 |
| **分层引导** | 至少 2 层（顶层 + Project/Topic选择） | 文档评审 |
| **Project列表来源** | 明确数据源（yaml文件） | 数据验证 |
| **示例完整** | 至少 2 个完整示例 | 用户测试 |

## 八、与原设计对比

| 改进项 | 原设计 v1.0 | 新设计 v2.0 |
|--------|-------------|-------------|
| **引导层级** | 单层（任务类型引导） | 三层（归属验证 → Project选择 → 任务类型） |
| **核心约束** | 无明确约束 | 三个"不能"约束 ⭐ |
| **顶层引导** | 无 | Project/Task归属验证 ⭐ |
| **Project列表** | 无 | 从yaml文件读取 ⭐ |
