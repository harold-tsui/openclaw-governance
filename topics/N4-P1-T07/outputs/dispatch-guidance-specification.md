> **版本**: v1.0
> **创建日期**: 2026-04-15
> **作者**: 张铁 (cqo)
> **用途**: 详细描述引导触发位置 + 引导内容类型

---

## 一、文档目的

**核心目的**：
1. 明确哪些地方需要引导
2. 明确每种引导需要什么样的内容

**约束原则**：
- 三个"不能"：不能出现未知的项目、不能出现孤儿的task card、不能出现孤儿的topic
- 引导选项 ≤ 5 个（避免认知负担）
- 明确归属不引导（提高效率）

## 二、引导触发位置清单

### 2.1 按意图类型分类

| 意图类型 | 是否需要引导 | 引导触发条件 | 引导位置 |
|----------|-------------|-------------|----------|
| **create_project** | ✅ 需要引导 | 用户想创建新Project | 层级 1（顶层） |
| **create_task** | ✅ 需要引导 | 用户想创建新Task，归属不明确 | 层级 1（顶层） |
| **create_topic** | ✅ 需要引导 | 用户想创建新Topic，归属不明确 | 层级 1（顶层） |
| **task_complete** | ✅ 需要引导 | Task完成，需要确认下一步 | 阶段转换引导 |
| **task_blocked** | ✅ 需要引导 | Task阻塞，需要确认处理方式 | 阶段转换引导 |
| **check_status** | ⚠️ 可能需要引导 | 检查范围不明确（全局/单项目/单Task） | 范围确认引导 |
| **archive_task** | ⚠️ 可能需要引导 | 归档位置不明确 | 归档位置引导 |
| **execute_task** | ❌ 不引导 | 归属明确（显式声明或state记录） | 直接执行 |

---

### 2.2 按任务生命周期阶段分类

| 阶段 | 是否需要引导 | 引导位置 | 引导内容 |
|------|-------------|----------|----------|
| **① 创建** | ✅ 需要引导（不明确归属） | 层级 1/2/3 | 确认归属 + Task类型 |
| **② 执行** | ❌ 不引导（明确归属） | - | 直接执行 |
| **③ 检查** | ⚠️ 可能需要引导 | 范围确认 | 检查范围（全局/单项目/单Task） |
| **④ 完成** | ✅ 需要引导 | 阶段转换 | 下一步动作（创建后续/归档/更新MISSION_BOARD） |
| **④ 阻塞** | ✅ 需要引导 | 阶段转换 | 处理方式（上报Harold/自行解决/等待外部） |
| **⑤ 归档** | ⚠️ 可能需要引导 | 归档位置 | 归档目录确认 |

---

### 2.3 按用户消息特征分类

| 用户消息特征 | 是否需要引导 | 引导位置 | 示例 |
|--------------|-------------|----------|------|
| **显式声明项目名称** | ❌ 不引导 | - | "继续 ZT-P015 的工作" |
| **显式声明Topic名称** | ❌ 不引导 | - | "继续 N4-P1-T07 的任务" |
| **无项目信息** | ✅ 需要引导 | 层级 1 | "帮我创建一个任务" |
| **dispatch-state.json 有记录** | ❌ 不引导 | - | 上次活跃项目记录存在 |
| **意图歧义（多个候选）** | ✅ 需要引导 | 意图澄清 | 可能是创建Task/执行Task/检查状态 |

## 三、引导内容类型详解

### 3.1 层级 1：顶层引导（Project/Task 归属验证）

**引导位置**：意图识别后，归属不明确时

**引导类型 1：创建Project引导** ⭐

**触发条件**：用户说"帮我创建一个新项目"

**引导内容**：
```
我理解您想创建新项目。请确认：

A. 在已有项目基础上创建新Task/Topic（推荐，避免重复）
B. 确实需要创建新Project（需要Harold审批）
C. 只是查询/讨论，不需要创建

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

**后续动作**：
- 用户选择 A → 进入层级 2（列出已有Project）
- 用户选择 B → 提示需要Harold审批，暂停创建
- 用户选择 C → 进入查询/讨论模式

---

**引导类型 2：创建Task引导** ⭐

**触发条件**：用户说"帮我创建一个新任务"，归属不明确

**引导内容**：
```
我理解您想创建新任务。请确认Task归属：

A. 选择已有Project + Topic（推荐）
B. 放入我的临时工作台（ZT-P-AGT-{agent}）
C. 放入NUCLEUS治理基础（ZT-P000）

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

**后续动作**：
- 用户选择 A → 进入层级 2（列出已有Project + Topic）
- 用户选择 B/C → 允许创建，标记为"待归属"

---

**引导类型 3：创建Topic引导** ⭐

**触发条件**：用户说"帮我创建一个新Topic"，归属不明确

**引导内容**：
```
我理解您想创建新Topic。请确认Topic归属：

A. 选择已有Project（推荐）
B. 放入我的临时工作台（ZT-P-AGT-{agent}）
C. 放入NUCLEUS治理基础（ZT-P000）

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

**后续动作**：
- 用户选择 A → 进入层级 2（列出已有Project）
- 用户选择 B/C → 允许创建，标记为"待归属"

---

**引导类型 4：意图不清晰引导** ⭐

**触发条件**：用户说"帮我处理一下"，意图不清晰

**引导内容**：
```
我理解您有需求，但需要确认是否需要创建任务：

A. 需要创建新任务（会进入Task归属引导）
B. 只需要查询/讨论（不创建Task）
C. 需要执行已有任务（继续未完成的Task）

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

**后续动作**：
- 用户选择 A → 进入层级 1（创建Task引导）
- 用户选择 B → 进入查询/讨论模式
- 用户选择 C → 列出未完成Task供选择

---

### 3.2 层级 2：已有Project/Topic选择引导

**引导位置**：层级 1 用户选择 A 后

**引导类型 5：Project选择引导** ⭐

**触发条件**：用户在层级 1 选择 A（选择已有Project）

**引导内容**：
```
请选择已有Project：

1. ZT-P015 NUCLEUS 4.0（NUCLEUS灵枢计划）
2. ZT-P009 NUCLEUS 2.0（治理体系）
3. ZT-P-AGT-cqo（张铁个人工作台）
4. 其他（列出更多）

回复数字或Project名称。
```

**选项数量**：≤ 5 个（活跃Project列表 + "其他"选项）

**数据源**：system-projects.yaml + user-projects.yaml

**排除规则**：
- 排除已关闭的Project（status ≠ active）
- 排除已归档的Project

---

**引导类型 6：Topic选择引导** ⭐

**触发条件**：用户选择Project后，需要选择Topic

**引导内容**：
```
请选择 ZT-P015 下的Topic：

1. N4-P1-T07 BotLearn方案
2. N4-P2-T01 CycleAggregator
3. 其他（列出更多）

回复数字或Topic名称。
```

**选项数量**：≤ 5 个（活跃Topic列表 + "其他"选项）

**数据源**：Project/TOPIC-BRIEF.md 或 Project目录扫描

**排除规则**：
- 排除已关闭的Topic
- 排除已归档的Topic

---

### 3.3 层级 3：任务类型引导（可选）

**引导位置**：层级 2 完成（Project + Topic归属明确）后

**引导类型 7：任务类型引导**

**触发条件**：归属明确，但任务类型不明确

**引导内容**：
```
已确认Task归属：ZT-P015/N4-P1-T07

请选择任务类型：

A. 创建新Task-CARD
B. 执行已有Task
C. 查看Task状态
D. 修改Task

回复 A/B/C/D 或描述具体需求。
```

**选项数量**：4 个（A/B/C/D）

---

### 3.4 阶段转换引导

**引导位置**：Task状态转换时（完成/阻塞）

**引导类型 8：任务完成 → 下一步引导** ⭐

**触发条件**：用户说"任务完成了"

**引导内容**：
```
任务已完成。请选择下一步：

A. 创建后续任务（在同一Project/Topic下）
B. 归档任务（移动到archive目录）
C. 更新 MISSION_BOARD（状态标记为已完成）
D. 暂无后续动作

回复 A/B/C/D 或描述具体需求。
```

**选项数量**：4 个（A/B/C/D）

**关键改进**：
- 用户选择 A → 自动继承上一个Task的Project + Topic归属 ⭐
- 避免重新引导归属，提高效率

---

**引导类型 9：任务阻塞 → 处理方式引导** ⭐

**触发条件**：用户说"任务阻塞了"或dispatch检测到阻塞

**引导内容**：
```
任务已阻塞。请选择处理方式：

A. 上报Harold（需要Harold决策）
B. 自行解决（尝试自己解决阻塞）
C. 等待外部条件（等待他人/外部条件）
D. 暂时搁置（标记为阻塞，稍后处理）

回复 A/B/C/D 或描述具体需求。
```

**选项数量**：4 个（A/B/C/D）

---

### 3.5 范围确认引导

**引导位置**：检查状态时，范围不明确

**引导类型 10：检查范围引导**

**触发条件**：用户说"检查任务状态"，范围不明确

**引导内容**：
```
请确认检查范围：

A. 全局检查（所有Project/Task）
B. 单项目检查（指定Project）
C. 单Task检查（指定Task）

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

---

### 3.6 归档位置引导

**引导位置**：归档Task时，位置不明确

**引导类型 11：归档位置引导**

**触发条件**：用户说"归档这个任务"，位置不明确

**引导内容**：
```
请确认归档位置：

A. Project archive目录（10_Projects/ZT-P015/archive/）
B. Topic archive目录（topics/N4-P1-T07/archive/）
C. 全局archive目录（.system/governance/archive/）

回复 A/B/C 或描述具体需求。
```

**选项数量**：3 个（A/B/C）

## 四、引导选项数量限制规则

**核心原则**：避免用户认知负担

| 引导层级 | 选项数量上限 | 说明 |
|----------|-------------|------|
| **层级 1（顶层）** | 3 个 | A/B/C 选项 |
| **层级 2（Project/Topic选择）** | 5 个 | 活跃列表 + "其他"选项 |
| **层级 3（任务类型）** | 4 个 | A/B/C/D 选项 |
| **阶段转换引导** | 4 个 | A/B/C/D 选项 |
| **范围确认引导** | 3 个 | A/B/C 选项 |

## 五、引导与状态文件关联

### 5.1 引导记录到 dispatch-state.json

**记录字段**：

| 字段 | 用途 | 更新时机 |
|------|------|----------|
| `user_preferences.last_choice` | 上次引导选择 | 用户选择后 |
| `user_preferences.choice_history` | 选择历史列表 | 每次选择后 |
| `session_state.active_project` | 当前活跃Project | Project选择后 |
| `session_state.active_topic` | 当前活跃Topic | Topic选择后 |
| `task_lifecycle.current_stage` | 当前任务阶段 | 阶段转换时 |

---

### 5.2 引导读取 dispatch-state.json

**读取字段**：

| 字段 | 用途 | 读取时机 |
|------|------|----------|
| `session_state.active_project` | 判断是否明确归属 | 每次引导前 |
| `guidance_templates` | 获取引导模板内容 | 输出引导时 |
| `user_preferences.last_choice` | 优先路由到上次选择 | 意图歧义时 |

## 六、引导流程完整示例

### 示例 1：完整三层引导流程

**用户**: "帮我创建一个任务"

**Step 1（层级 1）**:
```
我理解您想创建新任务。请确认Task归属：

A. 选择已有Project + Topic（推荐）
B. 放入我的临时工作台（ZT-P-AGT-cqo）
C. 放入NUCLEUS治理基础（ZT-P000）

回复 A/B/C 或描述具体需求。
```

**用户**: "A"

**Step 2（层级 2）**:
```
请选择已有Project：

1. ZT-P015 NUCLEUS 4.0
2. ZT-P009 NUCLEUS 2.0
3. ZT-P-AGT-cqo

回复数字或Project名称。
```

**用户**: "1"

**Step 3（层级 2 继续）**:
```
请选择 ZT-P015 下的Topic：

1. N4-P1-T07 BotLearn方案
2. N4-P2-T01 CycleAggregator

回复数字或Topic名称。
```

**用户**: "1"

**Step 4（层级 3）**:
```
已确认Task归属：ZT-P015/N4-P1-T07

请选择任务类型：

A. 创建新Task-CARD
B. 执行已有Task
C. 查看Task状态

回复 A/B/C 或描述具体需求。
```

**用户**: "A"

**Step 5（创建Task-CARD）**:
```
已确认：创建新Task-CARD，归属 ZT-P015/N4-P1-T07

请输入任务名称：
```

### 示例 2：显式声明（无引导）

**用户**: "我在继续进行 ZT-P015/N4-P1-T07 的工作"

**dispatch 解析**:
- 规则 1 匹配：显式声明项目 + Topic ✅
- 不需要引导

**执行动作**:
```
已确认：ZT-P015 NUCLEUS 4.0 / N4-P1-T07 BotLearn方案

请选择具体动作：
A. 执行已有Task
B. 创建新Task
C. 查看Topic状态

回复 A/B/C 或描述具体需求。
```

---

### 示例 3：任务完成后引导

**用户**: "任务完成了"

**dispatch 解析**:
- 读取 dispatch-state.json.task_lifecycle.current_stage = "执行中"
- 状态转换 → "完成"
- 需要引导下一步 ⭐

**引导内容**:
```
任务已完成。请选择下一步：

A. 创建后续任务（在同一Project/Topic下）
B. 归档任务（移动到archive目录）
C. 更新 MISSION_BOARD（状态标记为已完成）
D. 暂无后续动作

回复 A/B/C/D 或描述具体需求。
```

**用户**: "A"

**自动继承归属**:
```
已确认后续任务归属：ZT-P015/N4-P1-T07（继承上一个Task）

请输入任务名称：
```

## 七、验收标准

| 验收项 | 标准 | 验证方法 |
|--------|------|----------|
| **引导位置清单** | 11 种引导类型完整 | 文档审查 |

| **状态文件关联** | 记录/读取字段完整 | JSON schema 验证 |
