## 一、引导触发条件 ⭐

### 1.1 核心规则

| 场景 | 是否引导 | 示例 |
|------|---------|------|
| **明确的项目归属** | ❌ 不引导 | 用户说"继续 ZT-P015 的工作" → 直接执行 |
| **不明确的归属** | ✅ 需要引导 | 用户说"帮我创建一个任务" → 引导确认归属 |
| **可显式声明** | ✅ 支持 | 用户可说"我在继续进行 ZT-P015/N4-P1-T07 的工作" |

---

### 1.2 明确归属判断规则

**判断逻辑**：

```python
def is_explicit_project(user_message):
    """判断用户消息是否明确项目归属"""
    
    # 规则 1：用户显式声明项目名称
    if "ZT-P" in user_message or "NUCLEUS" in user_message:
        return True
    
    # 规则 2：用户说"继续某项目的工作"
    if "继续" in user_message and "工作" in user_message:
        # 提取项目名称或Topic名称
        return True
    
    # 规则 3：dispatch-state.json 记录了上次活跃项目
    if state.session_state.active_project is not None:
        # 用户没有明确切换项目 → 继续上次项目
        return True
    
    # 其他情况 → 不明确，需要引导
    return False
```

### 1.3 显式声明示例

**用户显式声明**：

| 用户消息 | dispatch 解析 | 执行动作 |
|----------|--------------|----------|
| "我在继续进行 ZT-P015 的工作" | active_project = ZT-P015 | 直接执行，不引导 |
| "继续 N4-P1-T07 的任务" | active_project = ZT-P015, active_topic = N4-P1-T07 | 直接执行，不引导 |
| "帮我创建一个任务" | 无明确归属 | 引导确认归属 ⭐ |

## 二、任务生命周期管理 ⭐

### 2.1 生命周期阶段

```
┌─────────────────────────────────────────┐
│         任务生命周期（从入口开始）          │
├─────────────────────────────────────────┤
│                                         │
│  ① 创建 → ② 执行 → ③ 检查 → ④ 完成/阻塞 → ⑤ 归档 │
│                                         │
└─────────────────────────────────────────┘
```

---

### 2.2 各阶段入口引导

| 阶段 | 用户意图示例 | 引导内容 | 是否需要引导 |
|------|--------------|----------|--------------|
| **① 创建** | "帮我创建一个任务" | 确认归属 + 类型 | ✅ 需要引导（不明确归属） |
| **② 执行** | "继续 ZT-P015 的工作" | 无需引导 | ❌ 直接执行（明确归属） |
| **③ 检查** | "检查任务状态" | 确认检查范围 | ⚠️ 可能需要引导（范围不明确） |
| **④ 完成/阻塞** | "任务完成了" | 确认下一步动作 | ✅ 需要引导（归档还是创建新任务） |
| **⑤ 归档** | "归档这个任务" | 确认归档位置 | ⚠️ 可能需要引导（归档目录） |

---

### 2.3 阶段转换引导

**阶段 ④ 完成 → 下一步引导**：

```
任务已完成。请选择下一步：

A. 创建后续任务（在同一Project/Topic下）
B. 归档任务（移动到archive目录）
C. 更新 MISSION_BOARD（状态标记为已完成）
D. 休息一下（暂无后续动作）

回复 A/B/C/D 或描述具体需求。
```

## 四、完整流程示例

### 4.1 明确归属（无引导）

**用户**: "我在继续进行 ZT-P015 的工作"

**dispatch 解析**：
- active_project = ZT-P015 ✅ 明确归属
- 不需要引导

**执行动作**：
```
已确认：ZT-P015 NUCLEUS 4.0

请选择具体动作：
A. 执行已有Task
B. 创建新Task
C. 查看项目状态

回复 A/B/C 或描述具体需求。
```

---

### 4.2 不明确归属（需要引导）

**用户**: "帮我创建一个任务"

**dispatch 解析**：
- 无明确项目归属 ❌ 不明确
- 需要引导 ⭐

**引导流程**：
```
我理解您想创建新任务。请确认Task归属：

A. 选择已有Project + Topic（推荐）
B. 放入我的临时工作台（ZT-P-AGT-cqo）
C. 放入NUCLEUS治理基础（ZT-P000）

回复 A/B/C 或描述具体需求。
```
