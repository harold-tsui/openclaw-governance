## 三、根因分析（CTO 已填写）

> **Step 1**：CTO 接收任务后，先进行根因分析，填写本章。

### 3.1 问题 1：CWD 状态不一致

| 维度 | 内容 |
|------|------|
| **根因** | `_setup()` 函数改变调用进程的 CWD，当被不同路径的 exec 调用时，`_original_cwd` 不一致 |
| **复现步骤** | 1. 从不同目录调用 `pdca.py` 2. 检查日志 `cwd` 字段 3. 观察路径变化 |
| **分析人** | 菡云芝 (CTO) |
| **分析时间** | 2026-04-20 11:00 |

**证据**：日志中 `cwd` 字段变化：
- `/Users/.../skills/openclaw-governance-nucleus`（正确）
- `/Users/.../60_Agents/cqo`（正确但不同）
- `/`（异常，根目录）

### 3.2 问题 2：exec 并行执行问题

| 维度 | 内容 |
|------|------|
| **根因** | LLM 在单个 reply 中同时发起多个 `pdca.py` 调用 → exec 并行执行 → Phase 校验失败 |
| **复现步骤** | 1. 在 heartbeat 中同时调用 p/d/c/a 2. 观察日志中的时间戳差异（毫秒级） |
| **分析人** | 菡云芝 (CTO) |
| **分析时间** | 2026-04-20 11:00 |

**证据**：日志显示同一 task 的命令在同一时间戳：
```
{"ts": "...14:12:30.806903", "cmd": "a", "ok": false}
{"ts": "...14:12:30.807130", "cmd": "c", "ok": true}
{"ts": "...14:12:30.809255", "cmd": "d", "ok": true}
```

### 3.3 问题 3：pdca.py 并发安全 bug

| Bug | 位置 | 问题描述 |
|-----|------|---------|
| **#1** | `_setup()` | CWD 设置后，后续命令使用相对路径可能失败 |
| **#2** | `_save()` | 并行写入时，临时文件 `.tmp` 可能冲突 |
| **#3** | `_current_cycle()` | 并行读取时，可能读到过时的 cycle 状态 |

| 分析人 | 菡云芝 (CTO) |
| 分析时间 | 2026-04-20 11:00 |
