## ❌ 冲突详情

### **冲突 1: Skills 引用路径格式不统一**

**问题描述**:  
不同 Skills 中对配置文件的引用路径格式不一致，可能导致加载逻辑混乱。

**具体表现**:
- **governance-hierarchy SKILL.md**: `config/system/agents.yaml` (相对路径)
- **governance-delegation SKILL.md**: `.system/governance/current/config/system/automation-levels.yaml` (绝对路径)  
- **governance-data SKILL.md**: `config/system/data-classification.yaml` (相对路径)

**实际文件位置**:  
`.system/governance/current/config/system/`

**修复建议**:  
统一所有 Skills 中的配置文件引用为**绝对路径格式**：
```
.system/governance/current/config/system/agents.yaml
.system/governance/current/config/system/automation-levels.yaml  
.system/governance/current/config/system/data-classification.yaml
```

---

### **冲突 2: 状态标记语义不一致**

**问题描述**:  
`[x]` 状态标记在不同文件中有不同含义，造成理解混乱。

**具体表现**:
- **governance-task SKILL.md 定义**: `[x]` = "**已完成**" (验收通过，已关闭)
- **MISSION_BOARD.md 实际使用**: 
  - `[x] 日常进行` (表示循环执行中)
  - `[x] 循环中` (表示持续运行)

**影响**:  
状态标记失去标准化意义，影响任务状态跟踪准确性。

**修复建议**:  
在 MISSION_BOARD.md 中，循环任务应使用 `[P]` (执行中) 状态，并添加说明：
- `[P] 循环中` - 表示周期性任务正在执行
- `[P] 日常进行` - 表示日常维护任务正在执行

保留 `[x]` 仅用于表示"**真正完成并关闭**"的任务。

---

### **冲突 3: AGENTS.md 内容与版本声明不符**

**问题描述**:  
AGENTS.md 的版本声明声称已更新按需加载表格，但实际内容未更新。

**具体表现**:
- **版本声明**: "v5.4 - 参考 IDENTITY.md v3.3，按需加载改为通过 Skill 间接引用"

**对比**:
- **IDENTITY.md v3.3**: 已正确更新为 Skill 间接引用表格
- **AGENTS.md v5.4**: 按需加载部分仍为空表格，未同步更新

**修复建议**:  
更新 AGENTS.md 的按需加载表格，使其与 IDENTITY.md v3.3 保持完全一致：

```markdown
| 任务类型 | 触发关键词 | 加载 Skill | 规范由 Skill 内部引用 |
|----------|------------|-----------|---------------------|
| Project管理 | 创建/关闭/审批 Project | governance-hierarchy | ZT-2026-001, ZT-2026-007 |
| Task/Topic管理 | 创建/关闭/分配 Task | governance-task | ZT-2026-005, ZT-2026-007 |
| 数据相关 | 数据分级/归档/合规 | governance-data | ZT-2026-004, ZT-2026-003 |
| 授权/Review | Review级别/授权降级 | governance-delegation | HAROLD-DELEGATION, ARCH-001 |
```
