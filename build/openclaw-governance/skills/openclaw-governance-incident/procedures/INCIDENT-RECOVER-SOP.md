# INCIDENT-RECOVER-SOP · 故障恢复流程

> **版本**：v1.0
> **触发条件**：故障确认后
> **审批级别**：L2（银月协调）→ L3（SEV-1 需要 Harold）
> **预计耗时**：根据故障级别

---

## 一、流程概览

```
Step 1: 响应团队组建 → Step 2: 影响评估 → Step 3: 恢复方案制定 → Step 4: 恢复执行 → Step 5: 验证确认 → Step 6: 故障关闭
```

---

## 二、响应团队

### 2.1 团队组成

| 级别 | 指挥人 | 团队成员 |
|------|--------|----------|
| **SEV-1** | Harold | 银月 + 相关 Agent + 外部支持 |
| **SEV-2** | 银月 | 相关 Agent |
| **SEV-3** | Agent 负责人 | 相关 Agent |
| **SEV-4** | Agent 负责人 | 相关 Agent |

### 2.2 角色

| 角色 | 职责 |
|------|------|
| **故障指挥** | 统筹协调、决策 |
| **技术负责** | 恢复方案设计、执行 |
| **沟通负责** | 状态同步、用户沟通 |
| **记录员** | 时间线记录、文档维护 |

---

## 三、详细步骤

### Step 1: 响应团队组建（预计 5-15min）

```
□ 确认故障指挥人
□ 召集响应团队
□ 分配角色
□ 建立沟通渠道
□ 输出：响应团队就绪
```

**时间要求**：
- SEV-1: 15 分钟内
- SEV-2: 30 分钟内
- SEV-3: 2 小时内
- SEV-4: 1 天内

---

### Step 2: 影响评估（预计 10-30min）

```
□ 确认影响范围
□ 评估用户影响
□ 评估数据影响
□ 评估业务影响
□ 输出：影响评估报告
```

**评估维度**：
- 影响系统/服务
- 影响用户数量和比例
- 数据丢失/损坏风险
- 业务损失估计
- 恢复时间预估

---

### Step 3: 恢复方案制定（预计 10-30min）

```
□ 分析故障原因
□ 制定恢复方案
□ 评估方案风险
□ 获得审批（SEV-1/SEV-2）
□ 输出：恢复方案
```

**方案要素**：
- 恢复步骤
- 预计时间
- 所需资源
- 风险评估
- 回滚方案

**审批要求**：
- SEV-1: 需要 Harold 审批
- SEV-2: 需要银月审批
- SEV-3/4: Agent 自主决定

---

### Step 4: 恢复执行（预计 变化）

```
□ 执行恢复步骤
□ 实时记录时间线
□ 同步状态更新
□ 处理意外情况
□ 输出：恢复完成
```

**执行原则**：
- 先恢复，后优化
- 记录每一步操作
- 及时同步状态
- 保留回滚选项

**时间线记录**：
```markdown
| 时间 | 操作人 | 操作 | 结果 |
|------|--------|------|------|
| 13:20 | main | 收到告警 | 开始调查 |
| 13:25 | main | 确认故障 | SEV-2 |
| 13:30 | cto | 加入响应 | 技术负责 |
| ... | ... | ... | ... |
```

---

### Step 5: 验证确认（预计 10-30min）

```
□ 执行功能验证
□ 确认服务恢复
□ 确认用户可访问
□ 监控稳定性
□ 输出：验证通过
```

**验证清单**：
- [ ] 核心功能正常
- [ ] 用户可正常使用
- [ ] 数据完整性确认
- [ ] 无新的异常告警
- [ ] 性能指标正常

---

### Step 6: 故障关闭（预计 5min）

```
□ 更新故障状态
□ 记录恢复时间
□ 通知相关人员
□ 触发复盘流程（如需要）
□ 输出：故障关闭
```

**关闭条件**：
- 服务已恢复
- 验证已通过
- 用户可正常使用

---

## 四、恢复目标时间

| 级别 | 响应时间 | 恢复目标 | 总时间 |
|------|----------|----------|--------|
| **SEV-1** | 15 分钟 | 1 小时 | < 2 小时 |
| **SEV-2** | 30 分钟 | 4 小时 | < 5 小时 |
| **SEV-3** | 2 小时 | 24 小时 | < 26 小时 |

---

## 五、状态流转

```
detected → confirmed → responding → recovering → recovered → verified → closed
    ↓           ↓            ↓             ↓            ↓
  (检测)    (确认)       (响应)        (恢复)       (验证)       (关闭)
```

| 状态 | 说明 |
|------|------|
| **detected** | 故障已发现 |
| **confirmed** | 故障已确认 |
| **responding** | 响应团队已组建 |
| **recovering** | 恢复执行中 |
| **recovered** | 恢复完成 |
| **verified** | 验证通过 |
| **closed** | 故障关闭 |
