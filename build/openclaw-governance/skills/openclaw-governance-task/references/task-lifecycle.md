## Task-Card 定义

**Task-Card 是工单**，是记录传递工作的最核心载体。

**职责**：记录一个工作从前因后果到结束关闭，以及关闭后知识沉淀的完整生命周期。

---

## 三区分层架构

| Zone | 性质 | 章节 | 更新时机 |
|------|------|------|----------|
| **Zone A** | 任务定义 | §一～§六 | 创建时填写，基本不变 |
| **Zone B** | 运行时状态 | §七～§八 | 执行中持续更新 |
| **Zone C** | 后处理 | §九 | 关闭后异步完成 |

## 状态流转

### 通用状态

| 状态 | 标记 | 含义 |
|------|------|------|
| 待接收 | `[ ]` | 已创建，等待 Task PIC 接收 |
| 执行中 | `[P]` | 已接收，执行中 |
| 待验收 | `[V]` | 已提交，等待验收 |
| 已完成 | `[x]` | 验收通过，已关闭 |
| 归档 | `[archived]` | 知识沉淀完成，已归档 |
| 阻塞 | `[?]` | 遇到阻塞，需解除后继续 |
| 待决策 | `[!]` | 需要 Harold 决策后继续 |

### 状态流转图

```
[ ] 待接收 ──accept──→ [P] 执行中 ──submit──→ [V] 待验收 ──approve──→ [x] 已完成
     │                     │                      │                          │
     │ reject              │ blocked              │ reject                   │
     ↓                     ↓                      ↓                          ↓
  回退分派            [?] 阻塞                 [P] 执行中            知识沉淀
                                                                       ↓
                                                               [archived] 归档
```

## 生命周期闭环

### 创建时闭环

| 步骤 | 动作 | 执行人 |
|------|------|--------|
| ① | 生成 Task-Card（Zone A 预填） | 指派人 |
| ② | 更新 MISSION_BOARD §五 待处理队列 | 指派人 |
| ③ | 通知 Task PIC | 指派人 |

### 接收时闭环

| 步骤 | 动作 | 执行人 |
|------|------|--------|
| ① | Task-Card §七.1 状态改为 `[P]` | Task PIC |
| ② | 更新 MISSION_BOARD §三 进行中区 | Task PIC |
| ③ | 通知指派人已接收 | Task PIC |

### 关闭时闭环

| 步骤 | 动作 | 执行人 |
|------|------|--------|
| ① | verify_task_closed() 全部通过 | 指派人 |
| ② | Task-Card §七.1 状态改为 `[x]` | 指派人 |
| ③ | 更新 MISSION_BOARD §六 已完成区 | 指派人 |
| ④ | 通知 Task PIC | 指派人 |
