# ALERT-SUMMARY-SOP · 告警汇总流程

> **版本**：v1.0
> **触发条件**：日报/周报/月报需求
> **审批级别**：L2（银月审核）
> **预计耗时**：15-30 分钟

---

## 一、流程概览

```
Step 1: 时间范围确定 → Step 2: 告警筛选 → Step 3: 统计分析 → Step 4: 报告生成 → Step 5: 分发确认
```

---

## 二、汇总类型

| 类型 | 时间范围 | 频率 | 接收人 |
|------|----------|------|--------|
| **日报** | 过去 24 小时 | 每日 | 银月 |
| **周报** | 过去 7 天 | 每周一 | 银月 + Harold |
| **月报** | 过去 30 天 | 每月 1 日 | Harold |
| **专项报告** | 指定时间段 | 按需 | 指定人员 |

---

## 三、详细步骤

### Step 1: 时间范围确定（预计 1min）

```
□ 确认汇总类型（日报/周报/月报/专项）
□ 设定时间范围
□ 确认接收人
□ 输出：汇总参数
```

**时间范围格式**：
- 日报：`{yesterday} 00:00` - `{yesterday} 23:59`
- 周报：`{last_monday} 00:00` - `{last_sunday} 23:59`
- 月报：`{first_day_of_last_month} 00:00` - `{last_day_of_last_month} 23:59`

---

### Step 2: 告警筛选（预计 5min）

```
□ 按时间范围筛选告警
□ 按类型分类
□ 按级别分组
□ 输出：告警列表
```

**筛选维度**：
- 时间范围
- 告警级别（P0/P1/P2/P3）
- 告警类型（系统/服务/业务/安全）
- 状态（已关闭/处理中/待处理）
- 来源系统

---

### Step 3: 统计分析（预计 10min）

```
□ 统计各级别告警数量
□ 计算平均响应时间
□ 分析告警趋势
□ 识别高频告警
□ 输出：统计数据
```

**统计指标**：
- 总告警数
- P0/P1/P2/P3 分布
- 平均响应时间
- 平均处理时间
- MTTR（平均恢复时间）
- 告警环比变化
- Top 5 高频告警

---

### Step 4: 报告生成（预计 10min）

```
□ 使用模板生成报告
□ 填充统计数据
□ 添加关键告警详情
□ 生成改进建议
□ 输出：告警汇总报告
```

**报告内容**：
1. 概览摘要
2. 统计数据
3. 关键告警分析
4. 趋势分析
5. 改进建议

---

### Step 5: 分发确认（预计 2min）

```
□ 按类型发送报告
□ 确认接收状态
□ 记录发送日志
□ 归档报告
```

**分发方式**：
- 日报：飞书消息
- 周报：飞书 + 归档文件
- 月报：飞书 + 文档 + 归档
