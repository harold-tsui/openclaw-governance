# ALERT-ESCALATION-SOP · 告警升级流程

> **版本**：v1.0
> **触发条件**：告警分级/超时未处理
> **审批级别**：L1→L2→L3（逐级升级）
> **预计耗时**：5-15 分钟

---

## 一、流程概览

```
Step 1: 分级评估 → Step 2: 响应计时 → Step 3: 自动升级检查 → Step 4: 手动升级处理 → Step 5: 通知发送
```

---

## 二、告警分级标准

### P0 - 紧急（Critical）

| 维度 | 标准 |
|------|------|
| **影响** | 核心业务完全中断 |
| **用户** | 全部用户无法使用 |
| **数据** | 数据丢失风险 |
| **响应** | 5 分钟内响应 |
| **升级** | 15 分钟无响应 → 升级 Harold |

**P0 示例**：
- 数据库完全不可用
- Agent 集群全部离线
- 核心服务宕机

---

### P1 - 高优先级（High）

| 维度 | 标准 |
|------|------|
| **影响** | 核心功能受损，部分不可用 |
| **用户** | 部分用户受影响 |
| **响应** | 15 分钟内响应 |
| **升级** | 1 小时无响应 → 升级银月 |

**P1 示例**：
- 单个 Agent 异常
- 部分服务降级
- 性能严重下降

---

### P2 - 中优先级（Medium）

| 维度 | 标准 |
|------|------|
| **影响** | 非核心功能异常 |
| **用户** | 少量用户受影响 |
| **响应** | 1 小时内响应 |
| **升级** | 4 小时无响应 → 升级银月 |

**P2 示例**：
- 非核心服务异常
- 可接受的性能下降
- 功能受限但不影响主流程

---

### P3 - 低优先级（Low）

| 维度 | 标准 |
|------|------|
| **影响** | 潜在风险，需关注 |
| **用户** | 暂无影响 |
| **响应** | 4 小时内响应 |
| **升级** | 24 小时无响应 → 记录 |

**P3 示例**：
- 资源使用率告警
- 日志异常模式
- 配置漂移

---

## 三、详细步骤

### Step 1: 分级评估（预计 2min）

```
□ 评估影响范围
□ 确定用户影响
□ 判断数据风险
□ 分配告警级别（P0-P3）
□ 输出：告警级别
```

**评估清单**：
- [ ] 影响范围：全部/部分/少量/无
- [ ] 核心功能：中断/受损/受限/无
- [ ] 数据风险：有/无
- [ ] 用户影响：全部/部分/少量/无

---

### Step 2: 响应计时（预计 1min）

```
□ 记录开始时间
□ 设定升级计时器
□ 分配责任人
□ 输出：计时开始
```

**计时规则**：
- P0: 5 分钟响应，15 分钟升级
- P1: 15 分钟响应，1 小时升级
- P2: 1 小时响应，4 小时升级
- P3: 4 小时响应，24 小时记录

---

### Step 3: 自动升级检查（持续）

```
□ 监控计时器
□ 检查处理状态
□ 超时触发升级
□ 输出：升级通知
```

**自动升级逻辑**：
```
if elapsed_time > response_timeout AND status != "processing":
    escalate(level + 1)
    notify(next_level_responder)
```

---

### Step 4: 手动升级处理

```
□ 处理人判断需要升级
□ 记录升级原因
□ 分配给更高级别处理人
□ 通知相关人员
```

**升级原因**：
- 超出处理人能力范围
- 需要更高级别权限
- 跨系统/跨团队协调
- 涉及 Harold 决策

---

### Step 5: 通知发送（预计 1min）

```
□ 根据级别选择通知渠道
□ 发送通知内容
□ 记录通知状态
□ 输出：通知完成
```

**通知内容**：
```yaml
alert_id: ALERT-2026-0331-001
level: P0
title: "[P0] Agent 心跳超时"
escalation: 自动升级
reason: 5 分钟无响应
assigned_to: Harold
timestamp: 2026-03-31T13:25:00+08:00
```

---

## 四、升级路径

```
L1（Agent 自动处理）
    ↓ 超时/无法处理
L2（银月协调）
    ↓ 超时/需要决策
L3（Harold 最终决策）
```
