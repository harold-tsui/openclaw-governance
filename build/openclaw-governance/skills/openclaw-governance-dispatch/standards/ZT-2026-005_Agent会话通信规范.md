## 三、通信方式

### 3.1 两种方式对比

| 方式 | 用途 | 前提条件 | 适用场景 |
|------|------|----------|----------|
| `sessions_send` | 向**已有活跃会话**的 agent 发消息 | 目标 agent 必须有活跃会话 | 轻量消息、复用现有会话 |
| `sessions_spawn` | **主动启动**目标 agent 会话 | 配置了 `allowAgents` | 需要目标 agent 执行任务 |

### 3.2 决策流程

```
需要联系某 Agent
    ↓
检查是否有活跃会话 (sessions_list)
    ↓
├── 有活跃会话 → sessions_send (轻量，复用现有会话)
│
└── 无活跃会话 → sessions_spawn (主动启动，无需等待)
```

### 3.3 sessions_send 使用方法

**前提条件**：目标 agent 必须有活跃会话

**sessionKey 格式**：
```
agent:<agentId>:<channel>:<type>:<id>
```

**示例**：
```
agent:main:feishu:direct:ou_xxx
agent:cqo:feishu:group:oc_xxx
```

### 3.4 sessions_spawn 使用方法

**当目标 agent 没有活跃会话时，主动启动她的会话：**

```json
{
  "task": "银月，请处理以下任务：[具体内容]",
  "agentId": "main",
  "mode": "run",
  "cleanup": "keep",
  "runTimeoutSeconds": 900
}
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `task` | 发送给目标 agent 的任务内容 |
| `agentId` | 目标 agent 的 ID（见对照表） |
| `mode` | `run` = one-shot，`session` = 持久会话 |
| `cleanup` | `keep` = 保留执行记录，`delete` = 自动删除 |
| `runTimeoutSeconds` | 超时时间（秒） |

## 四、配置要求

### 4.1 allowAgents 配置

每个 Agent 需要在 `openclaw.json` 中配置 `allowAgents`：

```json5
{
  "agents": {
    "list": [
      {
        "id": "<agentId>",
        "subagents": {
          "allowAgents": ["*"]  // 允许向任何 agent spawn
        }
      }
    ]
  }
}
```

**配置选项**：
- `["*"]` - 允许向任何 agent spawn
- `["main", "cqo"]` - 仅允许向特定 agent spawn

## 五、注意事项

1. **优先使用 sessions_send**：当目标 agent 有活跃会话时，优先使用 `sessions_send`，更轻量
2. **sessions_spawn 用于主动启动**：当目标 agent 没有活跃会话时，使用 `sessions_spawn` 主动启动，不需要被动等待
3. **保持 sessionKey 格式一致**：所有 Agent 使用统一的 sessionKey 格式
