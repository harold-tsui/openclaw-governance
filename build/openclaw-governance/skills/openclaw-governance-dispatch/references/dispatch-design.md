## 分层引导架构

```
┌─────────────────────────────────────┐
│ 层级 1：顶层引导（Project/Task归属验证）│ ← 最重要：三个"不能"
├─────────────────────────────────────┤
│ 层级 2：已有Project/Topic选择引导     │ ← 列出候选，避免未知
├─────────────────────────────────────┤
│ 层级 3：任务类型引导（如果需要）       │ ← 交付物/方法论/改进
└─────────────────────────────────────┘
```

## 11 种引导类型

### P0 优先级（关键引导）

1. **创建 Project 引导**：创建项目/新建项目/立项
2. **执行 Task 引导**：帮我完成任务/继续工作
3. **创建 Topic 引导**：创建专题/新建 Topic

### P1 优先级（重要引导）

4. **通用意图引导**：帮我处理/处理一下
5. **任务完成→下一步**：做完了/任务完成
6. **Project 选择引导**：在已有项目上创建
7. **Topic 选择引导**：选择 Topic

### P2 优先级（次要引导）

8. **检查范围引导**：检查状态/看看进度
9. **任务类型引导**：归属明确但类型不清

### P3 优先级（低频引导）

10. **任务阻塞→处理**：卡住了/阻塞了
11. **归档位置引导**：归档这个任务

## 引导触发条件

| 用户消息模式 | 触发层级 | 引导模板 |
|-------------|---------|---------|
| "创建一个新项目" | 层级 1 | §三.4.1 创建Project引导 |
| "创建一个新任务" | 层级 1 | §三.4.2 创建Task引导 |
| "处理一下文档"（意图模糊） | 层级 1 | §三.4.3 通用意图引导 |
| 归属明确 | 不引导 | §三.5 显式声明 → 执行 |
