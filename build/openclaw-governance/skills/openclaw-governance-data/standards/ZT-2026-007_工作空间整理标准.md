## 一、目的

定义掌天智能工作空间的整理规则，确保：
1. 数据有序存放
2. 熵积累可控
3. 垃圾及时清理
4. 未知项正确处理

---

## 三、分类处理规则

### 3.1 清理（CLEAN）— 必须删除

| 类型 | 识别特征 | 处理方式 |
|------|---------|---------|
| **macOS 系统文件** | `.DS_Store`, `._*` | 删除 |
| **编辑器临时文件** | `*~`, `*.swp`, `*.tmp` | 删除 |
| **Python 缓存** | `__pycache__/`, `*.pyc`, `*.pyo` | 删除（工作目录） |
| **Node 缓存** | `.node-*`, `node_modules/.cache/` | 删除 |
| **Git 合并冲突** | `*.orig`, `*_merge_*` | 删除 |
| **编译产物** | `*.o`, `*.class`, `*.so`（非 venv） | 删除 |
| **堆栈转储** | `*.stackdump`, `core` | 归档后删除 |
| **垃圾目录** | `~/~`, `backup/old/` | 确认后删除 |

**注意**：
- `venv/`, `.venv/`, `site-packages/` 中的 `__pycache__` **不清理**（会破坏环境）
- `node_modules/` 中的缓存**不清理**

---

### 3.2 移动（MOVE）— 重新归档

| 类型 | 识别特征 | 目标位置 |
|------|---------|---------|
| **已完成项目** | Project 状态 = `archived` | `40_Archives/[category]/[PROJECT_ID]/` |
| **过期配置** | `*.bak`, `*.old`, `*_v1.*` | `40_Archives/technical-debt/` |
| **旧版本文档** | `*_v1.*`, `*_deprecated.*` | `40_Archives/deprecated_[type]/` |
| **长日志文件** | `logs/` 中 > 30 天 | `40_Archives/` 或 `.system/backups/` |
| **过期会话** | `memory/` 中 > 7 天 | `.system/backups/` |
| **大型临时文件** | `> 100MB` 且无引用 | `.system/backups/` 或删除 |

---

### 3.3 保留（KEEP）— 正常工作

| 类型 | 说明 |
|------|------|
| **正在执行的项目** | `10_Projects/` 下 Active 项目 |
| **Agent 工作文件** | `60_Agents/` 下 IDENTITY/SOUL 等 |
| **系统治理文件** | `.system/governance/current/` |
| **Skills** | `skills/` 下所有技能 |
| **OpenClaw 配置** | `.openclaw/` 配置 |
| **向量数据** | `qdrant_data/` |

---

### 3.4 未知（UNKNOWN）— 需确认

| 类型 | 处理流程 |
|------|---------|
| **无法识别的目录/文件** | 1. 记录到 `memory/YYYY-MM-DD-unknown-items.md` |
| | 2. 通知 CDO 辛如音 |
| | 3. CDO 判断：清理 / 移动 / 保留 |
| | 4. 更新本标准的识别规则 |
| **命名异常的目录** | 同上，但优先联系创建者 Agent |

## 四、熵控制指标

| 指标 | 目标值 | 告警阈值 |
|------|--------|---------|
| **目录深度** | ≤ 5 层 | > 7 层 |
| **单目录文件数** | ≤ 100 | > 200 |
| **空目录** | 0 | > 0 |
| **临时文件比例** | ≤ 1% | > 5% |
| **memory 目录大小** | ≤ 500MB | > 1GB |
| **logs 目录大小** | ≤ 500MB | > 1GB |

## 六、异常处理

### 6.1 误删恢复
```
1. 检查 deleted/ 备份目录
2. 使用 restore_deleted.sh 恢复
3. 记录误删原因
4. 更新清理规则
```

### 6.2 无法移动的文件
```
1. 检查权限
2. 检查是否被占用
3. 检查磁盘空间
4. 记录到异常日志
5. 通知 CDO
```
