## 一、核心原则

> **精简优先**：只备份人为生产的数据，排除自动生成的数据。

| 原则 | 说明 |
|------|------|
| **生产数据优先** | 只备份人为生产的数据（项目文件、Agent 配置、治理文件） |
| **排除自动数据** | 排除安装的 skill、运行时数据、日志、缓存 |
| **分级备份策略** | 增量 30 天 + 全量 4 周 + 季度归档 + 年度归档 |

## 三、备份策略

### 3.1 备份类型与周期 ⭐ NEW

| 备份类型 | 执行频率 | 保留策略 | 说明 |
|---------|---------|---------|------|
| **增量备份** | 每天 | 30 天轮换 | 仅备份变更的生产数据 |
| **全量备份** | 每 7 天 | 4 周轮换（保留 4 份） | 完整备份所有生产数据 |
| **季度归档** | 每季度 | 3 年轮换 | 每季度末备份，保留最近 12 份（3 年） |
| **年度归档** | 每年 | 永久 | 每年保留最近 1 份季度归档 |

### 3.2 备份周期详解 ⭐

```
增量备份（每天执行，保留 30 天）：
  Day 1: incremental/2026-04-01/
  Day 2: incremental/2026-04-02/
  ...
  Day 30: incremental/2026-04-30/
  Day 31: incremental/2026-04-01/ ← 被覆盖（轮换）

全量备份（每 7 天执行，保留 4 周）：
  Week 1: full/2026-W14/        ← 第 1 份
  Week 2: full/2026-W15/        ← 第 2 份
  Week 3: full/2026-W16/        ← 第 3 份
  Week 4: full/2026-W17/        ← 第 4 份
  Week 5: full/2026-W14/        ← 被覆盖（轮换）

季度归档（每季度末执行，保留 3 年）：
  2026-Q1: quarterly/2026-Q1/   ← 第 1 份
  2026-Q2: quarterly/2026-Q2/   ← 第 2 份
  ...
  2029-Q1: quarterly/2029-Q1/   ← 第 12 份
  2029-Q2: quarterly/2026-Q1/   ← 被覆盖（轮换，保留 3 年）

年度归档（每年保留最近 1 份季度归档）：
  2026: yearly/2026-Q4/         ← 2026 年保留 2026-Q4
  2027: yearly/2027-Q4/         ← 2027 年保留 2027-Q4
  ...
```

### 3.3 备份目录结构

```
${OPENCLAW_BACKUP_WORKSPACE}/
├── incremental/                 # 增量备份（30 天轮换）
│   ├── 2026-04-01/
│   ├── 2026-04-02/
│   └── .../                     # Day 1-30
├── full/                        # 全量备份（4 周轮换）
│   ├── 2026-W14/
│   ├── 2026-W15/
│   ├── 2026-W16/
│   └── 2026-W17/                # Week 1-4
├── quarterly/                   # 季度归档（3 年轮换）
│   ├── 2026-Q1/
│   ├── 2026-Q2/
│   └── .../                     # 保留 12 份（3 年）
├── yearly/                      # 年度归档（永久）
│   ├── 2026/ → 2026-Q4/         # 2026 年归档
│   └── .../
├── deleted/                     # 删除前备份（90 天）
└── backup.log                   # 备份日志
```

## 五、备份脚本

### 5.1 增量备份脚本

```bash
#!/bin/bash
# backup_incremental.sh - 增量备份脚本（v4.0）

WORKSPACE="${OPENCLAW_LOCAL_WORKSPACE:-~/Workspaces/openclaw/main}"
BACKUP_ROOT="${OPENCLAW_BACKUP_WORKSPACE:-~/Workspaces/openclaw.bak}"
DATE=$(date +%Y-%m-%d)
BACKUP_DIR="${BACKUP_ROOT}/incremental/${DATE}"

mkdir -p "${BACKUP_DIR}"

# 使用 rsync 增量备份（仅备份变更的生产数据）
rsync -a --delete \
    --exclude='skills/' \
    --exclude='memory/' \
    --exclude='logs/' \
    --exclude='.system/logs/' \
    --exclude='.system/governance/heartbeat-logs/' \
    --exclude='.system/governance/logs/' \
    --exclude='qdrant_data/' \
    --exclude='__pycache__/' \
    --exclude='node_modules/' \
    --exclude='.cache/' \
    --exclude='.agent_workspace/' \
    --exclude='60_Agents/*/.openclaw/' \
    --exclude='60_Agents/*/agent/' \
    --exclude='60_Agents/*/inbox/' \
    --exclude='*.tmp' \
    --exclude='.DS_Store' \
    --exclude='.env' \
    --exclude='*.key' \
    "${WORKSPACE}/" "${BACKUP_DIR}/"

# 清理 30 天前的增量备份
find "${BACKUP_ROOT}/incremental/" -maxdepth 1 -type d -mtime +30 -exec rm -rf {} \;
```

### 5.3 季度归档脚本

```bash
#!/bin/bash
# backup_quarterly.sh - 季度归档脚本（v4.0）

WORKSPACE="${OPENCLAW_LOCAL_WORKSPACE:-~/Workspaces/openclaw/main}"
BACKUP_ROOT="${OPENCLAW_BACKUP_WORKSPACE:-~/Workspaces/openclaw.bak}"
YEAR=$(date +%Y)
QUARTER=$((($(date +%m) - 1) / 3 + 1))
ARCHIVE_NAME="${YEAR}-Q${QUARTER}"
ARCHIVE_DIR="${BACKUP_ROOT}/quarterly/${ARCHIVE_NAME}"

mkdir -p "${ARCHIVE_DIR}"

# 完整备份并压缩
rsync -a \
    --exclude='skills/' \
    --exclude='memory/' \
    --exclude='logs/' \
    --exclude='.system/logs/' \
    --exclude='.system/governance/heartbeat-logs/' \
    --exclude='.system/governance/logs/' \
    --exclude='qdrant_data/' \
    --exclude='__pycache__/' \
    --exclude='node_modules/' \
    --exclude='.cache/' \
    --exclude='.agent_workspace/' \
    --exclude='60_Agents/*/.openclaw/' \
    --exclude='60_Agents/*/agent/' \
    --exclude='60_Agents/*/inbox/' \
    --exclude='*.tmp' \
    --exclude='.DS_Store' \
    --exclude='.env' \
    --exclude='*.key' \
    "${WORKSPACE}/" "${ARCHIVE_DIR}/"

# 压缩归档
cd "${BACKUP_ROOT}/quarterly/"
tar -czf "${ARCHIVE_NAME}.tar.gz" "${ARCHIVE_NAME}/"
rm -rf "${ARCHIVE_DIR}"

# 清理 3 年前的季度归档
find "${BACKUP_ROOT}/quarterly/" -name "*.tar.gz" -mtime +1095 -exec rm -f {} \;
```

### 5.4 年度归档脚本

```bash
#!/bin/bash
# backup_yearly.sh - 年度归档脚本（v4.0）

BACKUP_ROOT="${OPENCLAW_BACKUP_WORKSPACE:-~/Workspaces/openclaw.bak}"
YEAR=$(date +%Y)
PREV_YEAR=$((YEAR - 1))

# 创建年度归档（硬链接到最近季度归档）
LATEST_QUARTER=$(ls -t "${BACKUP_ROOT}/quarterly/"*.tar.gz 2>/dev/null | head -1)
if [ -n "${LATEST_QUARTER}" ]; then
    mkdir -p "${BACKUP_ROOT}/yearly/${PREV_YEAR}/"
    cp "${LATEST_QUARTER}" "${BACKUP_ROOT}/yearly/${PREV_YEAR}/${PREV_YEAR}-archive.tar.gz"
fi
```

### 6.1 Git 备份命令

#!/bin/bash
# github_backup.sh - GitHub 备份脚本（v4.0）

cd "${OPENCLAW_LOCAL_WORKSPACE:-~/Workspaces/openclaw/main}"

# git add .（自动排除 .gitignore 中的文件）
git add .

# 检查是否有变更
if git diff --cached --quiet; then
    echo "[$(date)] No changes to commit"
    exit 0
fi

# 提交变更
git commit -m "Auto-backup: $(date +'%Y-%m-%d %H:%M:%S')"

# 推送到远程
git push origin main
