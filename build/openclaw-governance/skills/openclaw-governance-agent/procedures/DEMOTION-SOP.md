# DEMOTION-SOP · Agent 降级流程

> **版本**：v1.0
> **触发条件**：连续任务失败/错误率过高/能力不匹配/主动申请
> **审批级别**：L2/L3
> **预计耗时**：1-2 小时

---

## 一、降级类型

| 类型 | 说明 | 审批级别 |
|------|------|----------|
| **自动化级别降级** | L4 → L3 → L2 → L1 → L0 | L2 (银月 + Harold 知情) |
| **职责缩减** | 减少职责范围 | L2 |
| **权限回收** | 回收部分权限 | L3 (Harold) |

---

## 二、降级条件（自动化级别）

| 当前级别 | 错误率 | 连续失败次数 | 触发降级 |
|----------|--------|--------------|----------|
| L4 | >= 10% | >= 2 | → L3 |
| L3 | >= 15% | >= 3 | → L2 |
| L2 | >= 20% | >= 3 | → L1 |
| L1 | >= 30% | >= 5 | → L0 |

---

## 三、流程概览

```
Step 1: 问题识别 → Step 2: 降级评估 → Step 3: 审批流程 → Step 4: 降级实施 → Step 5: 降级记录
```

---

## 四、详细步骤

### Step 1: 问题识别

```
□ 监控指标异常（错误率、失败率）
□ 分析降级原因
□ 评估影响
```

---

### Step 2: 降级评估

```
□ 确认降级必要性
□ 确定降级幅度
□ 制定改进计划（可选）
```

---

### Step 3: 审批流程

```
□ 银月审核（L2）
□ Harold 知情/审批（视严重程度）
□ 权限回收需 Harold 审批（L3）
```

---

### Step 4: 降级实施

```
□ 更新 agents.yaml 级别
□ 回收相应权限
□ 调整任务分配
```

**交付物**：`DEMOTION-REPORT-{agent_id}-{date}.md`

---

### Step 5: 降级记录

```
□ 记录降级原因
□ 制定改进计划（如有）
□ 更新个人档案
```
