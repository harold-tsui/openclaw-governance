# PROMOTION-SOP · Agent 升级流程

> **版本**：v1.0
> **触发条件**：自动化级别评估达标/Agent 申请/银月推荐
> **审批级别**：L2/L3
> **预计耗时**：1-2 小时

---

## 一、升级类型

| 类型 | 说明 | 审批级别 |
|------|------|----------|
| **自动化级别升级** | L0 → L1 → L2 → L3 → L4 | L3 (Harold) |
| **职责扩展** | 新增职责领域 | L2 |
| **权限提升** | 获得更多系统权限 | L3 (Harold) |

---

## 二、升级条件（自动化级别）

| 目标级别 | DL 命中率 | 连续成功次数 | 错误率 |
|----------|-----------|--------------|--------|
| L1 | >= 60% | >= 2 | < 20% |
| L2 | >= 70% | >= 3 | < 15% |
| L3 | >= 80% | >= 5 | < 10% |
| L4 | >= 90% | >= 10 | < 5% |

---

## 三、流程概览

```
Step 1: 升级评估 → Step 2: 升级申请 → Step 3: 审批流程 → Step 4: 升级实施
```

---

## 四、详细步骤

### Step 1: 升级评估

```
□ 检查自动化级别指标
  - DL 命中率
  - 连续成功次数
  - 错误率
□ 评估是否满足升级条件
```

---

### Step 2: 升级申请

```
□ 填写升级申请
□ 提供支持数据
□ 提交银月初审
```

**交付物**：`PROMOTE-REPORT-{agent_id}-{date}.md`

---

### Step 3: 审批流程

```
□ 银月审核（L2）
□ Harold 审批（L3）- 自动化级别/权限提升
□ 记录审批意见
```

---

### Step 4: 升级实施

```
□ 更新 agents.yaml 级别
□ 更新权限配置
□ 通知相关 Agent
□ 记录到个人档案
```
