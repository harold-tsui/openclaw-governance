# TRANSFER-SOP · Agent 调岗流程

> **版本**：v1.0
> **触发条件**：业务需求变更/Agent 能力匹配调整/组织架构调整
> **审批级别**：L2
> **预计耗时**：2-4 小时

---

## 一、调岗类型

| 类型 | 说明 | 审批级别 |
|------|------|----------|
| **职责调整** | 调整职责范围 | L2 |
| **项目调动** | 调整负责项目 | L2 |
| **协作关系变更** | 调整汇报对象/协作对象 | L1 |

---

## 二、流程概览

```
Step 1: 调岗需求识别 → Step 2: 调岗计划 → Step 3: 交接执行 → Step 4: 配置更新 → Step 5: 调岗确认
```

---

## 三、详细步骤

### Step 1: 调岗需求识别

```
□ 识别调岗原因
□ 确定调整内容
□ 评估影响范围
```

---

### Step 2: 调岗计划

```
□ 制定交接计划
□ 确定时间节点
□ 通知相关方
```

---

### Step 3: 交接执行

```
□ 知识交接（文档、任务）
□ 权限交接
□ 项目交接
```

---

### Step 4: 配置更新

```
□ 更新 IDENTITY.md
□ 更新 agents.yaml
□ 更新项目配置
```

**交付物**：`TRANSFER-REPORT-{agent_id}-{date}.md`

---

### Step 5: 调岗确认

```
□ 银月审核确认
□ 更新个人档案
□ 通知 Harold
```
