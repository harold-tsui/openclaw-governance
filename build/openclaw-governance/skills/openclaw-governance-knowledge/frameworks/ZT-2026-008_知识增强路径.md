## 完整流程

```
Task 执行
    ↓
知识沉淀（Wiki + DL 条目）
    ↓
知识索引（分类、标签、关键词）
    ↓
知识检索（新任务启动时自动检索相关知识）
    ↓
知识复用（参考历史经验、避免重复劳动）
    ↓
工作效率提升（减少试错、加快决策）
    ↓
新的知识沉淀（迭代优化）
```

## 知识复用机制

### 1. 自动检索触发点

| 触发点 | 检索内容 | 应用场景 |
|--------|---------|---------|

## 九、洞察内飞轮模型（v1.1 新增）

> **目标**：让"洞察"成为可运转的飞轮，而非偶尔的灵光一现。

### 9.1 洞察模型结构

```yaml
洞察模型（Insight Model）:
  输入源:
    - 外部情报（CIO 例行采集）
    - 内部产出（DL/LL/Wiki）
    - 环境变化（市场/技术/政策）
    - 思维碎片（[洞察] 标记）
  
  洞察类型:
    - 机会洞察：外部变化带来的新机会
    - 风险洞察：潜在威胁和风险
    - 模式洞察：从重复经验中提取的规律
    - 空白洞察：尚未被覆盖的领域
    
  洞察评估:
    - 时效性：当前/短期/长期
    - 置信度：高/中/低
    - 行动建议：立即行动/观察/归档
    
  输出:
    - 关联现有 DL（增强决策支撑）
    - 生成新 LL（如果发现错误假设）
    - 更新 Wiki 标签（价值变化标记）
    - 触发新 Task（如果需要行动）
```

### 9.2 文档价值动态评估

随着时间迁移，外部环境变化，文档价值也会发生变化。

| 评估维度 | 说明 | 触发条件 |
|---------|------|---------|
| 时效性 | 内容是否仍然有效 | 每季度 / 重大环境变化时 |
| 完整性 | 是否有新的相关信息可补充 | 日常洞察时检查 |
| 关联性 | 与其他文档的关系是否变化 | 新 DL/LL 产生时 |
| 置信度 | 假设前提是否仍然成立 | LL 触发时 |

**评估流程**：
```
CIO 执行日常洞察 → 扫描已有文档 → 评估价值变化 → 
  → 标记过时内容（更新 tags: stale）
  → 补充新洞察（生成洞察条目）
  → 触发 LL（如发现假设错误）
```

### 9.3 日常洞察执行规范（CIO 每日 16:00）

#### 执行时间
每日 16:00（可调整）

#### 扫描范围
| 范围 | 内容 |
|------|------|
| 新增 Wiki | 过去 24 小时创建的 Wiki |
| 新增 DL | 过去 24 小时创建的 DL 条目 |
| 新增 LL | 过去 24 小时触发的 Lesson Learn |
| 思维碎片 | 带 `[洞察]` 标签的 Feishu 消息 |
| 外部情报 | CIO 例行采集的最新情报 |

#### 评估维度（快速扫描）
- 是否有新洞察？
- 文档价值是否有变化？
- 是否有重大外部环境变化需要标记？

#### 输出：《每日洞察速报》

```markdown
# 每日洞察速报 · {YYYY-MM-DD}

## 今日洞察
| 洞察类型 | 内容摘要 | 关联文档 | 建议 |
|---------|---------|---------|------|
| 机会洞察 | ... | DL-XXX | 观察/行动 |
| 风险洞察 | ... | Wiki-XXX | 观察 |

## 价值变化标记
| 文档 | 变化类型 | 原因 |
|-----|---------|------|
| DL-XXX | 置信度下降 | 市场环境变化 |

## 明日关注
- 话题 1
- 话题 2
```

### 9.4 深度洞察执行规范（每周/每两周）

#### 执行频率
每周或每两周一次（视情况而定）

#### 联动数据
| 数据源 | 用途 |
|--------|------|
| 外部情报 | 行业报告、竞品动态、政策变化 |
| DL 命中率 | 分析决策质量趋势 |
| LL 触发情况 | 发现系统性问题 |
| Wiki 访问频次 | 识别高价值知识 |
| Project 进展 | 关联业务结果 |

#### 输出：《深度洞察报告》

```markdown
# 深度洞察报告 · {YYYY-MM-DD}

## 本期主题
{主题名称}

## 模式发现
- 模式 1
- 模式 2

## 假设验证
| 假设 | 验证结果 | 置信度 |
|-----|---------|-------|
| ... | ... | ... |

## 行动建议
| 建议 | 优先级 | 关联 Task |
|-----|-------|----------|
| ... | P0 | Task-XXX |

## 与现有 DL/LL 的关联
- DL-XXX: ...
- LL-XXX: ...
```

### 9.5 与 DL/LL/Wiki 联动规则

#### 洞察 → DL 增强
- 日常洞察中发现与现有 DL 相关的内容 → 补充到 DL 的"延伸适用"字段
- 发现新的决策模式 → 生成新 DL 草稿

#### 洞察 → LL 触发
- 深度洞察中发现假设错误 → 触发 LL 流程
- 标记可能过时的 DL（置信度下降）

#### 洞察 → Wiki 更新
- 文档价值变化 → 更新 Wiki 的 `tags` 字段（添加 `stale` / `needs-review`）
- 发现新关联 → 更新 `related-docs` 字段

#### 洞察 → Task 触发
- 需要采取行动 → 生成新 Task-CARD
- 重大发现 → 通知 Harold
