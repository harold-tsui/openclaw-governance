## P0-P3 定义

| 优先级 | 名称 | 定义 | 响应时限 |
|:------:|------|------|---------|
| **P0** | 紧急重要 | 影响核心目标达成，必须立即处理 | 立即处理 |
| **P1** | 重要不紧急 | 对目标达成有重要影响，需要尽快处理 | 6小时内 |
| **P2** | 一般任务 | 常规工作，可以在空闲时处理 | 24小时内 |
| **P3** | 待观察 | 价值不确定，可以暂缓 | 有空再做 |

---

## 典型场景示例

| 优先级 | 场景 |
|--------|------|
| **P0** | 系统崩溃、安全漏洞、严重阻塞 |
| **P1** | 核心功能开发、重要会议准备、关键交付物 |
| **P2** | 代码优化、文档完善、常规维护 |
| **P3** | 新功能调研、兴趣爱好类任务 |

---

## 快速决策矩阵

```
问题：这项任务应该是什么优先级？

1. 不做会直接影响核心目标吗？
   → 是 → P0

2. 不做会影响近期重要里程碑吗？
   → 是 → P1

3. 有明确的价值但不太紧急吗？
   → 是 → P2

4. 价值不确定或是探索性的吗？
   → 是 → P3
```
