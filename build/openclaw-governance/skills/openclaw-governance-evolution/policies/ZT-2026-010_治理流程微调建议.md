# ZT-2026-007 · 治理流程微调建议 v1.0

> **关联文件**：ZT-2026-005 工作流程与约束规则 v2.0、ZT-2026-006 组织架构规范 v1.0
> **调整原因**：明确 PM 汇报机制、银月角色边界、治理遵循要求

---

## 一、核心调整

### 1.1 PM 独立汇报机制

| 调整项 | 当前规则 | 建议调整 |
|--------|---------|---------|
| **汇报路径** | 混合在 main session | PM 通过飞书 channel 直接向 Harold 汇报 |
| **Session 区分** | 无明确区分 | 每个 Project 独立 Feishu thread/channel |
| **汇报内容** | 混合在日常对话 | Project 级进度、Topic 级状态、Task 级 blocker |

**飞书 channel 命名建议**：
```
Harold - {PM} - {ProjectID}
例：Harold - CTO - ZT-P005
```

### 1.2 银月 (PA) 角色边界

| 维度 | 调整后职责 |
|------|-----------|
| **Project 级** | 总体进度跟踪、项目级优先级决策、跨项目资源协调 |
| **Topic/Task 级** | 不直接管理，PM 负责 |
| **协调权限** | 项目间冲突、项目优先级排序、Harold 指令传达 |

**职责边界**：
- ✅ 银月可以：检查 Project 进度、催办、协调优先级
- ❌ 银月不应：直接创建/关闭 Topic/Task、干预 PM 执行细节

### 1.3 PM 职责明确化

| 职责 | 说明 |
|------|------|
| **项目交付** | 对 Project 整体交付负责 |
| **独立汇报** | 通过独立 Feishu channel 向 Harold 汇报 |
| **Topic 管理** | 创建/管理 Topic 和 Task |
| **进度跟踪** | 维护自己项目的 MISSION_BOARD |
| **资源协调** | 需要时向银月申请协调 |

### 1.4 治理遵循要求

所有 Agent（包括 subagent）必须遵循 `.system/governance` 中的规范：

| 规则层级 | 说明 |
|---------|------|
| **Policies** | 约束性规范（如 ZT-2026-005 工作流程） |
| **Procedures** | 流程规范（如 PROJECT-LIFECYCLE） |
| **Templates** | 交付物模板 |
| **Decisions** | 架构决策记录（ADR） |

## 二、Session 管理建议

### 2.1 Session 分类

| Session 类型 | 说明 | 管理者 |
|-------------|------|--------|
| **Main Session** | Harold 与 银月的日常沟通 | 银月 |
| **PM Session** | PM 单独向 Harold 汇报 | PM 各自 |
| **Project Session** | Project 执行专用 | PM |
| **Admin Session** | 系统管理、执行命令 | 银月 |

### 2.2 飞书 Channel 策略

每个 Project 建议独立 channel：
- 避免 session 混杂
- 便于追溯项目历史
- 明确 PM 职责边界

## 五、Sub Agent Onboarding 规范

### 5.1 Main Agent 校验规则

> **核心原则**：未完成 onboarding 的 Agent 不能接收正式任务

| 检查项 | 说明 |
|--------|------|
| **onboarding_status** | `persons.yaml` 中的 `onboarding_status` 字段 |
| **必需文件** | IDENTITY.md, SOUL.md, MISSIONBOARD.md, 目录结构 |
| **状态验证** | Main Agent 派发任务前必须校验 |

**校验流程**：
```
派发任务 → 检查 onboarding_status → 未完成 → 先执行 onboarding → 完成后可接任务
```

### 5.2 Onboarding 标准流程

| 步骤 | 动作 | 负责人 | 输出 |
|------|------|--------|------|
| 1 | 创建 onboarding task | 银月 | inbox/ONBOARDING.md |
| 2 | 初始化标准文件 | 银月 | IDENTITY.md, MISSIONBOARD.md, 目录 |
| 3 | 执行 onboarding 学习 | Sub Agent | 掌握治理规范 |
| 4 | **最后动作：自我介绍** | Sub Agent | 给 Harold 发送介绍消息 |

### 5.3 Onboarding 完成标志

**必须全部满足**：
1. ✅ 所有标准文件已创建（IDENTITY.md, SOUL.md, MISSIONBOARD.md, 目录结构）
2. ✅ onboarding_status = "completed" 在 persons.yaml
3. ✅ **Agent 给 Harold 发送自我介绍消息**（最后一个动作）

### 5.4 自我介绍消息模板

Agent 完成 onboarding 后，必须向 Harold 发送以下内容：

```
👋 你好！我是 {agent_name}（{role}）

我已经完成 onboarding，现在可以正式工作了！

我的职责：{core_responsibilities}
我的汇报对象：银月 → Harold
我的工作目录：{workspace_path}

期待为你服务！🚀
```
