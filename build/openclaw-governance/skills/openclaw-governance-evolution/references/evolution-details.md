# Evolution PDCA — Detailed Reference

> Moved from evolution SKILL.md to reduce main file size.

## §二 PDCA 评估框架

### Skill 质量指标采集

| 指标 | 来源 | 采集方式 |
|------|------|---------|
| 错误率 | governance-quality | 审查问题数/交付物数 |
| 使用频率 | governance-heartbeat | 加载次数/周期 |
| 版本迭代速度 | SKILL.md 版本历史 | 版本数/月 |
| 依赖稳定性 | dependency graph | 下游 skill 报错次数 |

### 评估等级

| 等级 | 条件 | 行动 |
|------|------|------|
| **优秀** | 错误率 < 2%，迭代稳定 | 维持现状 |
| **良好** | 错误率 2-5%，偶有问题 | 观察改进 |
| **需改进** | 错误率 > 5% 或用户反馈差 | 启动 PDCA 改进 |
| **待废弃** | 长期未使用/功能重复 | 标记 deprecated |

## §三 版本升级决策

### SemVer 规则

| 变更类型 | 版本增量 | 示例 |
|---------|---------|------|
| 架构重构、Skill 合并/拆分 | Major (X.0.0) | v2 → v3 |
| 新功能、新函数、新协议 | Minor (X.Y.0) | v2.1 → v2.2 |
| Bug 修复、文档更新 | Patch (X.Y.Z) | v2.1.0 → v2.1.1 |

## §四 架构改进

### 改进类型

| 类型 | 说明 | 输出 |
|------|------|------|
| **新 Skill** | 新增独立能力 | 新 SKILL.md + 注册表更新 |
| **Skill 合并** | 减少职责重叠 | 合并后 SKILL.md + 废弃旧文件 |
| **Skill 废弃** | 功能不再需要 | 标记 deprecated + 迁移指南 |
| **层级调整** | 调整加载优先级 | 更新 governance-core 注册表 |

## §五 质量门控

| 门控项 | 阈值 | 触发行动 |
|--------|------|---------|
| 错误率 > 10% | 高 | 立即启动 PDCA |
| SKILL.md > 500 行 | 中 | 拆分 references/ |
| 依赖数 > 5 | 中 | 评估依赖合理性 |
| 连续 30 天未使用 | 低 | 标记待废弃 |

## §六 执行流程

```
用户请求系统评估
    ↓
采集各 Skill 质量指标
    ↓
评估等级判定
    ↓
生成改进建议
    ↓
版本升级决策
    ↓
执行架构改进
    ↓
更新注册表和文档
```
