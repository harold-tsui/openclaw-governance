# 闭环治理 Topic 模板 · v2.0

> **用途**：创建闭环治理问题的 Topic
> **适用场景**：需要多步骤解决、涉及推广、需要持续跟踪的问题
> **维护人**：张铁 (CQO)

---

## 一、核心原则

> **闭环！闭环！闭环！**

问题的解决不是终点，推广、验收、持续跟踪才是完整的闭环。

---

## 二、闭环流程图

```
发现问题
    ↓
┌─────────────────────────────────────────────┐
│ 初始评估                                     │
│ 问题是否需要闭环治理？                        │
│   ├─ 否 → 简单修复 → 关闭（结束）             │
│   └─ 是 → 继续                               │
└─────────────────────────────────────────────┘
    ↓
创建 Topic（记录问题）
    ↓
规划 Task（根据问题特点选择需要的 Task）
    ↓
┌─────────────────────────────────────────────┐
│ (a) 试点                                     │
│     ↓                                        │
│ (b) 单点测试                                 │
│     ↓                                        │
│ ┌─────────────────────────────────────────┐ │
│ │ 决策：是否需要推广？                      │ │
│ │   ├─ 否 → 跳到 (e)                       │ │
│ │   └─ 是 → 继续                           │ │
│ └─────────────────────────────────────────┘ │
│     ↓                                        │
│ (c) 推广（可选）                             │
│     ↓                                        │
│ (d) 推广验收（可选）                         │
│     ↓                                        │
│ ┌─────────────────────────────────────────┐ │
│ │ 决策：是否需要持续跟踪？                  │ │
│ │   ├─ 高风险 → 必须跟踪                   │ │
│ │   ├─ 中风险 → 建议跟踪                   │ │
│ │   └─ 低风险 → 可选                       │ │
│ └─────────────────────────────────────────┘ │
│     ↓                                        │
│ (e) 持续跟踪（可选）                         │
│     ↓                                        │
│ (f) 采样结果判断                             │
│   ├─ 发现问题 → 回到 (a) 重新试点           │
│   └─ 无问题 → 继续                          │
│     ↓                                        │
│ (g) 关闭所有问题                             │
│     ↓                                        │
│ (h) 关闭 Topic                               │
└─────────────────────────────────────────────┘
```

| Task ID | 名称 | 触发条件 | 产出物 |

## 四、决策点说明

### 4.1 初始评估

| 问题类型 | 处理方式 | 示例 |
|----------|----------|------|
| 简单问题 | 直接修复，不创建 Topic | 配置错误、typo |
| 中等问题 | 创建 Topic，少量 Task | 流程优化 |
| 复杂问题 | 创建 Topic，完整 Task 集 | 多 Agent 影响的治理问题 |

### 4.2 是否推广

| 情况 | 决策 | 说明 |
|------|------|------|
| 问题只影响一个对象 | 不推广 | 只修复试点对象 |
| 问题影响多个对象 | 推广 | 修复所有相关对象 |
| 问题可能影响其他对象 | 观察期 | 先观察，再决定 |

### 4.3 是否持续跟踪

| 风险级别 | 决策 | 跟踪周期 |
|----------|------|----------|
| 高（数据安全、合规） | 必须跟踪 | 每日/每周 |
| 中（流程、效率） | 建议跟踪 | 每周/每月 |
| 低（体验优化） | 可选 | 按需 |
